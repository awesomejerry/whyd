## Context

WHYD 應用目前僅支援滑鼠操作，缺乏鍵盤快捷鍵支援。使用者需要更快速的方式來執行常見操作，特別是在頻繁記錄和查詢的工作流程中。應用使用原生 JavaScript，無外部框架依賴。

## Goals / Non-Goals

**Goals:**
- 提供全域鍵盤快捷鍵支援
- 實作幫助面板顯示所有快捷鍵
- 提供視覺回饋確認快捷鍵觸發
- 避免在輸入框中觸發快捷鍵

**Non-Goals:**
- 可自訂快捷鍵（未來功能）
- 快捷鍵序列（如 vim 的多鍵組合）
- 遊戲手把或其他輸入裝置支援

## Decisions

### 1. 事件監聽架構
**決定**: 使用單一全域 `keydown` 事件監聽器，集中處理所有快捷鍵
**替代方案**: 
- 為每個快捷鍵註冊獨立監聽器（缺點：效能較差，難以管理）
**理由**: 集中管理更容易維護，可統一處理輸入框排除邏輯

### 2. 修飾鍵處理
**決定**: 使用 `event.metaKey || event.ctrlKey` 處理跨平台支援
**理由**: Mac 使用 Cmd (metaKey)，Windows/Linux 使用 Ctrl (ctrlKey)

### 3. 輸入框排除
**決定**: 檢查 `event.target` 是否為 `INPUT`、`TEXTAREA` 或 `[contenteditable]`
**理由**: 避免在使用者輸入時意外觸發快捷鍵

### 4. 幫助面板實作
**決定**: 使用 Modal 覆蓋層，以 HTML 內建元素實作
**理由**: 與現有 Export Modal 模式一致，無需額外依賴

### 5. 視覺回饋
**決定**: 使用 CSS flash 動畫效果
**理由**: 輕量、無需 JavaScript 動畫庫，與現有樣式系統一致

## Risks / Trade-offs

**[Risk] 快捷鍵衝突**: 瀏覽器內建快捷鍵可能與應用快捷鍵衝突
→ 使用 `event.preventDefault()` 阻止預設行為，並在幫助面板中清楚說明

**[Risk] 無障礙問題**: 快捷鍵可能影響螢幕閱讀器使用者
→ 確保所有功能仍可透過滑鼠/觸控操作，快捷鍵為輔助功能

**[Risk] 跨瀏覽器差異**: 不同瀏覽器處理修飾鍵方式可能不同
→ 在主流瀏覽器（Chrome、Firefox、Safari）測試
